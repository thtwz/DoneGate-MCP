# DoneGate v0.2 Supervision Reporting Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add git-based supervision reporting so DoneGate can detect repository work that is not tied to an active task.

**Architecture:** Compute supervision state as a read model from the working tree and active task context, then persist the latest snapshot into `.donegate-mcp/supervision.json`. Expose the read model through a CLI command without changing lifecycle rules.

**Tech Stack:** Python 3.11+, argparse CLI, git status porcelain output, pytest.
