# DoneGate v0.2 Active Task Context Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add repository-local active task context and let bundled hooks fall back to it when `TASK_ID` is not explicitly provided.

**Architecture:** Persist a small `session.json` file under `.donegate-mcp/` and expose it through service and CLI methods. Keep hook logic thin by resolving the task id through the CLI rather than reimplementing state lookup in shell.

**Tech Stack:** Python 3.11+, argparse CLI, file-backed state, bash hooks, pytest.
